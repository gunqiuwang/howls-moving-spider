# 视觉方向说明：宫崎骏式童话机械世界

> 本文件用于指导本项目的视觉改造。目标是借鉴宫崎骏 / 吉卜力作品中的气质：温暖、手工感、旧机械、生活气、风与光。不要复刻具体电影镜头、角色或资产。

---

## 1. 一句话目标

保留当前八足移动城堡玩法，让画面从“程序化 3D 原型”推进到：

> 一个像活在童话山野中的旧机械移动城堡。

它应该笨重、荒诞、温柔、有生活痕迹，而不是冷硬机甲。

---

## 2. 视觉关键词

### 要靠近

- 温暖黄昏 / 清晨光
- 柔和天空和云
- 风吹草地
- 旧铁皮、老木头、暗红瓦
- 暖窗光、烟囱、炉火感
- 手工拼接、补丁、歪斜房屋
- 像有人居住的移动城堡

### 要远离

- 赛博朋克
- 军武机甲
- 冷白灯光
- 高反射金属
- 纯黑重工业怪兽
- 恐怖蜘蛛生物
- 默认 Three.js demo 感

---

## 3. 主体视觉：先像城堡，再像机械

当前项目保留 8 条腿，但用户第一眼应该先看到“移动城堡”，再注意到它有八足机械结构。

### 应强化

- 房屋体块不规则堆叠。
- 屋顶、烟囱、窗户、露台、木梁。
- 前部有脸感或头部识别点。
- 城堡像被多年修补过。
- 腿和身体连接可信，但不要比房屋更抢眼。

### 应避免

- 主体像完整装甲壳。
- 远看只是蜘蛛机器人。
- 腿部过于锋利、攻击性太强。
- 机械细节压过生活细节。

---

## 4. 材质语言

材质不追求写实 PBR，而追求“温暖、旧、手工”的统一观感。

### 推荐色彩

- 锈铁：暖棕、焦褐、旧铜感。
- 木头：米褐、深木色、有年轮或木板感。
- 瓦片：暗红棕，不要鲜红。
- 石头：温灰褐，不要现代混凝土。
- 窗光：暖黄、橙黄、柔和发光。

### 材质目标

- 有岁月感。
- 有修补感。
- 有使用痕迹。
- 没有现代塑料感。

---

## 5. 光照和空气感

这是第一轮视觉改造最重要的部分。

### 目标

- 场景更温暖。
- 阴影更柔和。
- 远景有空气透视。
- 天空和地面颜色协调。
- 画面有风、有光、有童话感。

### 建议方向

- 主光偏金黄。
- 半球光保留天空蓝 / 地面暖色关系。
- 雾色偏浅蓝或暖灰蓝。
- 避免过暗、过硬、过冷。
- 让窗光和炉火成为局部暖色焦点。

---

## 6. 环境氛围

环境不只是装饰，它要承担世界观。

### 应强化

- 草地颜色更自然。
- 云更柔软、绘本化。
- 村庄更温暖、更小巧。
- 风车、水井、围栏、帽子店有生活气。
- 路径、池塘、灌木不要像随机摆件。

### 理想效果

用户会觉得这个世界可以慢慢逛，而不是只用来测试控制器。

---

## 7. UI 方向

UI 应从“调试信息”变成“童话机械仪表”。

### 推荐方向

- 半透明暖棕面板。
- 黄铜边框。
- 米白 / 暖金文字。
- 任务提示像冒险目标。
- 按钮像黄铜铭牌或旧机械按钮。

### 避免

- 纯工程调试感。
- 颜色和画面脱节。
- 大块高对比现代 UI。

---

## 8. 改造原则

1. 先改整体气质，再补细节。
2. 先改光照、天空、材质、轮廓，再改粒子和小装饰。
3. 不推翻 8 足 IK 和现有玩法。
4. 不引入复杂新系统。
5. 每轮改动必须能解释视觉收益。

---

## 9. 视觉验收问题

视觉改造后，请逐项自问：

1. 第一眼更像移动城堡了吗？
2. 画面更温暖、更童话了吗？
3. 主体有旧机械和生活气了吗？
4. 环境像一个可漫游的童话世界了吗？
5. UI 是否不再像调试面板？
6. 是否仍然保留当前玩法可用性？
