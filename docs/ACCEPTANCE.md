# v0.3 视觉改造阶段验收标准

> 项目：哈尔风格八足移动城堡
> 当前目标：保留玩法基线，让视觉更接近宫崎骏 / 吉卜力式童话机械世界。

---

## 1. 主线范围

### 必须保留

- 8 足机械移动城堡。
- 行走、跳跃、飞行。
- 金币收集与任务提示。
- 中文 UI。
- 程序化 Three.js 实现。

### 当前不做

- 不改成电影原版双足。
- 不复刻具体电影镜头、角色、资产。
- 不做赛博朋克、军武机甲、冷科幻。
- 不引入大型外部 3D 模型。
- 不扩展复杂新玩法。

---

## 2. 工程验收

验收命令：

```bash
npm install
npm run build
npm run preview
```

通过标准：

- `npm install` 成功。
- `npm run build` 退出码为 0。
- `dist/` 正常生成。
- `npm run preview` 可启动。
- 浏览器打开后无初始化 fatal error。

已知本地问题：

- 如果出现 `@rollup/rollup-win32-x64-msvc` 缺包，先删除 `node_modules` 后重新安装依赖。

## 3. 功能验收

### 桌面端

- `W/A/S/D` 可移动。
- 鼠标拖拽可看视角。
- `Space` 可跳跃。
- `Shift` 可加速。
- `F` 可起飞 / 降落。
- `R` 或重置按钮可重置。
- 金币可收集，分数和剩余数量会变化。

### 移动端

- 虚拟摇杆可移动。
- 右侧拖拽可看视角。
- 加速、跳跃、起飞按钮可用。
- 中文按钮显示正确：起飞 / 降落。
- 连续游玩 2 分钟不崩溃。

---

## 4. 视觉验收

本阶段看的是“气质是否成立”，不是是否复刻电影。

### 通过标准

#### 主体识别

- 第一眼更像移动城堡，而不是普通蜘蛛机器人。
- 能看出屋顶、烟囱、窗户、房屋堆叠。
- 有旧铁皮、老木头、暗红瓦、暖窗光。
- 8 条腿与城堡主体连接可信，但不压过主体。

#### 气质统一

- 画面更温暖、更柔和、更童话。
- 有旧机械、手工感、生活气。
- 不像默认 3D demo。
- 没有明显赛博朋克、军武机甲、冷科幻感。

#### 环境氛围

- 天空、云、草地、村庄、风车、水井、围栏更协调。
- 远景有空气感。
- 世界像可漫游的童话山野。

#### UI 一致性

- HUD 不像纯调试面板。
- 任务提示与金币显示更像冒险目标。
- 按钮和文字配色与世界观统一。

### 不通过情况

- 主体仍第一眼像蜘蛛机器人。
- 画面偏冷、偏硬、偏科幻。
- 城堡缺少房屋群和生活气。
- 环境仍像测试场景。
- UI 明显跳戏。
- 视觉改造破坏行走、飞行、收集玩法。

---

## 5. 性能验收

### 桌面端

- 正常场景帧率大致稳定。
- 操控无明显延迟。
- 连续游玩 3 分钟不崩溃。

### 移动端

- 普通手机能基本操作。
- 连续游玩 2 分钟不崩溃。
- 触屏输入无严重卡死。

若视觉细节导致卡顿，应优先降低粒子、树木、草丛数量或 pixelRatio。

---

## 6. 提交前检查

每次 Codex 改完后必须输出：

- 修改文件列表。
- 改动目的。
- 视觉变化说明。
- 是否影响核心玩法。
- `npm run build` 结果。
- 浏览器控制台是否仍有 error。

建议提交信息：

```txt
style: improve Ghibli-inspired visual direction
```

或按具体改动拆分：

```txt
style: warm up world lighting and atmosphere
style: improve walking castle silhouette
style: align hud with storybook steampunk theme
```
