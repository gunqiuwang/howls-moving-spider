# 哈尔的移动城堡 — 完整重设计方案

> 方案状态：未来备选研究，不作为当前 v0.3 baseline 的直接执行路线。
> 当前主线保留 8 足机械移动城堡玩法；本文件只记录“电影原版双足城堡 / Toon 化重构”的可能方案。
> 若未来决定执行本方案，需要先单独立项，不应在 v0.3 基线收口阶段混入。

## 研究发现

### 电影原版城堡关键设计（非蜘蛛）
电影里的城堡只有 **两条鸟一样的机械腿**，不是蜘蛛的8条。它们以不可思议的方式支撑着庞大的建筑群。城堡身体是一堆**违章建筑拼凑体**：烟囱、吊臂、穹顶、望远镜、好几栋完整的房子被胡乱钉在一起，很多部件看起来锈迹斑斑。正面有**两个望远镜状的眼睛**和一个**机械大嘴**。

### 色彩规则
- **60:30:10 法则**：60% 天空蓝、30% 绿色山丘、10% 城堡暖色
- **Mattgyver 5色调色板**：用深饱和色代替纯黑，不用渐变，严格纯色块
- **从不使用纯黑**，用深棕/深紫代替

### Three.js 吉卜力渲染方法
- `MeshToonMaterial` + 梯度贴图（3级/5级色阶）→ 卡通色块化渲染
- 梯度贴图必须设 `NearestFilter` 才能产生硬边界色阶
- 半球光（天空蓝 + 地面暖色）模拟吉卜力柔和打光
- 不需要描边后处理——色阶本身就够了

## 一、腿系统：2条鸟形机械腿

### 现状
- 8条蜘蛛腿，每条有 femur + tibia + tarsus（轮子）
- 交替三角步态，IK 解算

### 改为
- **2条腿**，左右对称，从城堡底部两侧伸出
- 每条腿结构：
  ```
  髋关节（球关节，从城堡底部侧面伸出）
    → 上腿（粗壮，圆柱）
    → 膝关节（球关节，向后弯曲）
    → 下腿（较细，圆柱）
    → 踝关节（球关节）
    → 脚掌（3趾机械爪，向前三叉）
  ```
- **步态**：左右交替，身体左右摇摆重心转移
  - 左脚抬起 → 身体右倾 → 左脚前迈 → 落地 → 换右脚
  - 行走时城堡有明显的上下弹跳（bouncy walk cycle）
- **IK**：每条腿仍用 2-bone IK，但 gait 逻辑完全不同
- **飞行模式**：双腿向后伸直，像鸟滑翔

### 数据结构变化
```js
// 从 this.legs[8] 改为 this.legs[2]
this.legs = [
  { side: -1, hip: V3(-1.5, -1, 0), ... }, // 左腿
  { side:  1, hip: V3( 1.5, -1, 0), ... }, // 右腿
];
// 新增步态状态
this.walkPhase = 0;      // 0~1，左脚0-0.5，右脚0.5-1
this.walkBounce = 0;     // 身体弹跳偏移
this.weightShift = 0;    // 身体左右偏移
```

## 三、世界环境：吉卜力风格

### 天空
- Canvas 渐变：浅蓝 #87CEEB → 深蓝 #4A90D9
- 朵状白云（sprite billboard，缓慢飘动）
- 雾色：淡蓝 #B0D4E8

### 地形
- 柔和起伏的绿色山丘（现有的 fbm 噪声合适）
- 顶点着色：嫩绿 #6DB356 → 深绿 #4A8C3A → 山顶暖黄 #C8B870
- **不用纹理贴图**，纯顶点色 + toon 色阶 → 吉卜力手绘感

### 植被
- 圆锥形树冠（低多边形 6-8 面），3种绿色随机
- 树干：深棕，细长
- 散布在山坡上，避开城堡路径

### 村庄（远处，飞行时可见）
- 小方块房屋 + 尖屋顶
- 暖黄色墙壁 + 红色/棕色屋顶
- 不需要门窗细节（远景）

### 粒子
- 烟囱蒸汽（白色 sprite，上升消散）
- 行走时脚底尘土（棕色 sprite）
- 卡西法火焰粒子（城堡内部/窗户，红橙色）

## 四、渲染管线

### 材质系统
```js
// 创建 toon 梯度贴图（程序化 canvas）
function makeGradientMap(steps) {
  // steps = 3 → [暗, 中, 亮] 三档色阶
  const c = document.createElement('canvas');
  c.width = steps; c.height = 1;
  const x = c.getContext('2d');
  for (let i = 0; i < steps; i++) {
    const v = (i / (steps - 1)) * 255;
    x.fillStyle = `rgb(${v},${v},${v})`;
    x.fillRect(i, 0, 1, 1);
  }
  const t = new THREE.CanvasTexture(c);
  t.minFilter = THREE.NearestFilter;
  t.magFilter = THREE.NearestFilter;
  return t;
}

// 使用
const threeTone = makeGradientMap(3);
const mat = new THREE.MeshToonMaterial({
  color: 0x7A5A3A,
  gradientMap: threeTone,
});
```

### 灯光
```js
// 半球光：天空蓝 + 地面暖
scene.add(new THREE.HemisphereLight(0x87CEEB, 0x8B7355, 1.0));
// 主方向光（太阳）
const sun = new THREE.DirectionalLight(0xFFF4E0, 1.2);
sun.position.set(40, 50, 30);
```

### 雾
```js
scene.fog = new THREE.Fog(0xB0D4E8, 100, 300);
```

## 六、风险

1. **双足平衡**比8足难很多——需要身体重心偏移 + 侧倾补偿
2. **MeshToonMaterial** 不支持 flatShading——需要用 toon 梯度贴图模拟
3. 性能：toon 材质 + 更多几何体可能影响低端设备
